# update-readme

Update README so documentation matches the current implementation.

## Goal
Keep project documentation accurate, clear, and recruiter-friendly.

## Input (provide in prompt)
- Current README content (or relevant sections)
- What was implemented/changed
- Any new endpoints, services, or setup steps
- Screenshots/demo updates (if any)

## Output format
1. README sections to update
2. Inaccurate claims to remove/fix
3. New content to add
4. Updated feature list
5. Updated setup/run steps
6. Updated API/architecture summary
7. Demo flow updates
8. Final revised README text (only changed sections or full file)

## Rules
- Do not claim unimplemented features
- Use clear, concrete wording
- Keep terminology consistent across services
